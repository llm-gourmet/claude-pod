<objective>
Fix the sudo install.sh chown bug. When `sudo bash install.sh` runs, all files/dirs created under `$CONFIG_DIR` (~/.claude-secure/) are root-owned, causing the real user to hit "Permission denied" reading the chmod 600 .env.

Purpose: Restore the install flow so post-sudo-install the invoking user owns their own config dir.
Output: install.sh patched with two surgical chown calls.
</objective>

<task type="auto">
  <name>Task 1: Add chown calls to install.sh for invoking user ownership</name>
  <files>install.sh</files>
  <action>
    Two surgical edits to install.sh:

    **Edit 1** — Inside `setup_workspace()` (around line 327), immediately after `mkdir -p "$ws_path"`, add:
    ```bash
    chown "$_invoking_user:$_invoking_user" "$ws_path"
    ```
    This handles the workspace dir where `$ws_path` is a local variable and thus not reachable from main().

    **Edit 2** — In `main()` (around line 617-618), between `install_git_hooks` and `install_webhook_service`, insert:
    ```bash
    # Reclaim ownership of CONFIG_DIR for the invoking user (sudo creates as root).
    # install_cli + install_git_hooks + install_webhook_service deliberately leave
    # system paths (/usr/local/bin, /etc/systemd, /opt/claude-secure) as root.
    chown -R "$_invoking_user:$_invoking_user" "$CONFIG_DIR"
    chmod 777 "$CONFIG_DIR/logs"
    ```

    The `chmod 777` on logs is re-applied because `chown -R` preserves modes but we want the permission to be explicit and guaranteed after the ownership change (three containers write as UIDs 0/1000/1001).

    Do NOT add chown for `/usr/local/bin/claude-secure`, `/opt/claude-secure/`, or `/etc/systemd/system/*` — these are system paths and must remain root-owned.

    Non-sudo case is safe: when `SUDO_USER` is unset, `_invoking_user=$USER` and the files are already owned by that user, so `chown -R user:user` is a no-op.
  </action>
  <verify>
    <automated>bash -n install.sh && grep -q 'chown -R "$_invoking_user:$_invoking_user" "$CONFIG_DIR"' install.sh && grep -q 'chown "$_invoking_user:$_invoking_user" "$ws_path"' install.sh && grep -q 'chmod 777 "$CONFIG_DIR/logs"' install.sh</automated>
  </verify>
  <done>

    - chmod 777 on logs re-applied after chown
    - No chown added to /usr/local/bin, /opt/claude-secure, or /etc/systemd paths
  </done>
