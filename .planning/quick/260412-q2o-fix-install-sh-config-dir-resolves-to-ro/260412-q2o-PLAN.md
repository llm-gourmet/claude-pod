<objective>
Fix install.sh so that running it with sudo places all user-facing artifacts in the invoking user's home directory instead of /root.

Purpose: When an operator runs `sudo ./install.sh`, $HOME is /root. Config files, the workspace, and the fallback CLI path all land under /root/.claude-secure/ instead of the real user's home — making the installation unusable without manual cleanup.

Output: install.sh with invoking-user home resolution applied early (before any function reads CONFIG_DIR) and propagated to the workspace prompt default and the ~/.local/bin fallback path.
</objective>
