<objective>
Fix the empty `.env` bug in `install.sh setup_auth`, eliminate silent-failure paths, and bring `README.md` into exact alignment with what the installer actually does on the host. The user reported that after `sudo bash install.sh`, `~/.claude-secure/profiles/default/.env` was empty — their OAuth token was lost. Two separate defects combine into a bad first-run experience:

1. **setup_auth has silent failure paths.** `read -rsp` followed by `echo "KEY=${token}" > env_file` does not validate that the token is non-empty, does not require a TTY, and does not distinguish between "user pressed Enter by accident" / "paste buffer had a leading newline" / "stdin is not a TTY at all". In every broken case the result is a .env file that looks fine on disk but has no usable credential, and subsequent `claude-secure --profile default` calls silently fail to authenticate. `bin/claude-secure setup_profile_auth` is a near-identical copy of the same code and has the same bugs — fix both.

2. **README.md does not match reality.** README line 16 instructs `sudo ./install.sh` but `install.sh` is tracked in git as mode `100644` (non-executable), so on a fresh clone that command fails with `Permission denied` — which is why the reporting user fell back to `sudo bash install.sh` in the first place. README line 22 claims the installer "writes security hooks to /etc/claude-secure/" but the installer only writes `/etc/claude-secure/webhook.json` (and only when `--with-webhook` is used); the actual PreToolUse hook script lives inside the Docker image (`claude/hooks/pre-tool-use.sh`, COPY'd to `/etc/claude-secure/hooks/` in the container at build time, not on the host).

Purpose: Fresh `sudo ./install.sh` on a new machine must either succeed end-to-end with a valid .env OR abort with a clear, actionable error — never silently leave the user with broken credentials. README must be a precise description of what install.sh does, not a marketing paraphrase.

Output: Hardened install.sh, matching hardening in bin/claude-secure setup_profile_auth, corrected README.md.
