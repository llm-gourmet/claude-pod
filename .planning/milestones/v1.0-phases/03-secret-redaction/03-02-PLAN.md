Create `tests/test-phase3.sh` following the established test pattern from test-phase1.sh and test-phase2.sh.

**Test strategy:** The proxy forwards to `REAL_ANTHROPIC_BASE_URL`. For testing, override this to point to a simple echo server that reflects the request body and headers back. Use a Node.js one-liner started inside the proxy container as a mock upstream, or use a dedicated test approach. The simplest approach:

1. Start a mock upstream HTTP server inside the proxy container on port 9999 that echoes back the received request body and headers as JSON response
2. Temporarily set `REAL_ANTHROPIC_BASE_URL=http://localhost:9999` on the proxy (or use an env var override)
3. Send requests through the proxy containing secret values
4. Verify the mock upstream received placeholders (not real secrets)
5. Verify the client received restored values (not placeholders)

**Alternative simpler approach (preferred):** Since we can't easily swap env vars on a running container, use the proxy's own logging. Add a `/__test/echo` endpoint to the proxy that:
- Accepts a POST body
- Applies redaction (same as normal flow)
- Returns the redacted body directly (no upstream forwarding)
This test endpoint should ONLY work when `NODE_ENV=test` is set.

**Actually, the simplest approach:** The tests should:
1. Rebuild containers with test env vars set (e.g., `GITHUB_TOKEN=test_secret_gh_12345`)
2. Use `docker compose exec` to send requests from inside the claude container to the proxy
3. Since the proxy will try to forward to Anthropic and fail (no real API key), check the proxy logs for what was sent upstream
4. OR: Override `REAL_ANTHROPIC_BASE_URL` to a mock server

**Best approach for testing (use this):**
Create a small mock upstream server script `tests/mock-upstream.js` that runs inside the proxy container. The test script:

a. Copies `tests/mock-upstream.js` into the proxy container
b. Starts it in the background on port 9999
c. Sends test requests through the proxy with `REAL_ANTHROPIC_BASE_URL=http://localhost:9999`
d. The mock server writes received requests to a file that the test can read
e. Compares what the mock received (should have placeholders) vs what the client received (should have real values)

**However, environment variables are baked at container start.** The cleanest approach:

Create the test with docker compose override. The test script:

1. Creates a temporary `docker-compose.test-phase3.yml` override that:
   - Sets `REAL_ANTHROPIC_BASE_URL=http://localhost:9999` on proxy
   - Sets test secret values: `GITHUB_TOKEN=ghp_test_secret_value_12345`, `STRIPE_KEY=sk_test_stripe_secret_67890`, `OPENAI_API_KEY=sk-test-openai-secret-abcde`
   - Sets `ANTHROPIC_API_KEY=test-proxy-api-key-xyz`

2. Starts the mock upstream server inside proxy container:
   ```
   docker compose exec -d proxy node -e "
     const http = require('http');
     const fs = require('fs');
     http.createServer((req, res) => {
       let body = '';
       req.on('data', c => body += c);
       req.on('end', () => {
         const record = JSON.stringify({body, headers: req.headers}) + '\n';
         fs.appendFileSync('/tmp/upstream-requests.log', record);
         // Return a response containing a placeholder to test restoration
         res.writeHead(200, {'content-type': 'application/json'});
         res.end(JSON.stringify({reply: 'Your token is PLACEHOLDER_GITHUB and key is PLACEHOLDER_STRIPE'}));
       });
     }).listen(9999, () => console.log('Mock upstream on :9999'));
   "
   ```

3. Runs test cases:

**Test SECR-01: Proxy intercepts traffic**
- From claude container: `curl -s -o /dev/null -w '%{http_code}' http://proxy:8080/v1/messages`
- Expect: HTTP response (200 or any non-connection-error), proving proxy intercepts

**Test SECR-02: Secret values replaced with placeholders in outbound requests**
- From claude container: `curl -s -X POST http://proxy:8080/v1/messages -H 'content-type: application/json' -d '{"content":"my token is ghp_test_secret_value_12345"}'`
- Read `/tmp/upstream-requests.log` from proxy container
- Verify the logged request body contains `PLACEHOLDER_GITHUB` and does NOT contain `ghp_test_secret_value_12345`

**Test SECR-03: Placeholders restored to real values in responses**
- The mock upstream returns `{"reply":"Your token is PLACEHOLDER_GITHUB and key is PLACEHOLDER_STRIPE"}`
- From claude container: capture the response body from the curl above
- Verify response contains `ghp_test_secret_value_12345` and does NOT contain `PLACEHOLDER_GITHUB`
- Verify response contains `sk_test_stripe_secret_67890` and does NOT contain `PLACEHOLDER_STRIPE`

**Test SECR-04: Config hot-reload**
- Modify whitelist.json inside the proxy container (add a new secret entry with placeholder `PLACEHOLDER_HOTRELOAD` and env_var `HOTRELOAD_SECRET`)
- Set HOTRELOAD_SECRET env var won't work mid-container... So instead: modify an existing entry's placeholder
- Better approach: Test that the proxy re-reads the file by verifying behavior changes when the file changes. Since we can't change env vars mid-container, test this indirectly:
  - Send a request, verify redaction works (baseline)
  - Remove a secret entry from whitelist.json inside the container
  - Send same request, verify that secret is no longer redacted (because entry was removed)
  - Restore whitelist.json

**Test SECR-05: Auth credentials forwarded correctly**
- Read `/tmp/upstream-requests.log` from proxy container
- Verify the logged request headers contain `x-api-key: test-proxy-api-key-xyz` (the proxy's own key)
- Verify the logged request headers do NOT contain the claude container's dummy auth
