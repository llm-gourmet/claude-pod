<objective>
Make docker-compose.yml and all three log-producing services multi-instance ready by removing hardcoded container names, parameterizing the whitelist path, and adding LOG_PREFIX support for instance-prefixed log filenames.

Purpose: This is the foundation that enables COMPOSE_PROJECT_NAME-based isolation. Without removing container_name directives, a second instance will fail with "container name already in use." Without LOG_PREFIX, log files from multiple instances will overwrite each other.

Output: docker-compose.yml with no container_name directives and LOG_PREFIX/WHITELIST_PATH env vars; proxy.js, validator.py, pre-tool-use.sh updated to use LOG_PREFIX in log filenames.
</objective>

Update all three log-producing services to prepend `LOG_PREFIX` env var to their log filenames per D-07 and D-13.

**proxy/proxy.js** (line 11):
Change:
```javascript
const LOG_PATH = '/var/log/claude-secure/anthropic.jsonl';
```
To:
```javascript
const LOG_PREFIX = process.env.LOG_PREFIX || '';
const LOG_PATH = `/var/log/claude-secure/${LOG_PREFIX}anthropic.jsonl`;
```

**validator/validator.py** (line 367):
Change:
```python
json_handler = JsonFileHandler("/var/log/claude-secure/iptables.jsonl")
```
To:
```python
log_prefix = os.environ.get("LOG_PREFIX", "")
json_handler = JsonFileHandler(f"/var/log/claude-secure/{log_prefix}iptables.jsonl")
```
Also update the info log on line 369 to match:
```python
logger.info(f"JSON file logging enabled: /var/log/claude-secure/{log_prefix}iptables.jsonl")
```
