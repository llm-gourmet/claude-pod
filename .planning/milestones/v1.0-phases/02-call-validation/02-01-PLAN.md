Replace the stub validator.py with a full implementation. The validator is a Python HTTP server using only stdlib modules (http.server, sqlite3, socket, subprocess, threading, json, datetime, os, logging).

**Constants:**
- `DB_PATH = "/data/validator.db"`
- `CALL_TTL_SECONDS = 10`
- `LISTEN_PORT = 8088`
- `WHITELIST_PATH = "/etc/claude-secure/whitelist.json"`
- `CLEANUP_INTERVAL_SECONDS = 5`

**Database (init_db function):**
```sql
CREATE TABLE IF NOT EXISTS calls (
    call_id TEXT PRIMARY KEY,
    domain TEXT NOT NULL,
    ip_address TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    expires_at TEXT NOT NULL,
    used INTEGER NOT NULL DEFAULT 0
);
CREATE INDEX IF NOT EXISTS idx_expires ON calls(expires_at);
CREATE INDEX IF NOT EXISTS idx_used ON calls(used);
```
Use `PRAGMA journal_mode=WAL;` and `PRAGMA busy_timeout=5000;` on every connection.

**get_db() function:** Returns a new per-call sqlite3.Connection with WAL mode and Row factory. Do NOT share connections across threads.

**resolve_domain(domain) function:** Use `socket.getaddrinfo(domain, 443, socket.AF_INET)` to resolve domain to IPv4 address. Return the IP string or None on failure. For DNS resolution, since the validator shares claude's namespace (dns: 127.0.0.1), try Docker's embedded DNS at 127.0.0.11 by temporarily creating a socket connection. Implementation: use `socket.setdefaulttimeout(5)` and catch `socket.gaierror`. If resolution fails, log a warning and return None.

**add_iptables_rule(ip, call_id) function:** Run:
```
iptables -I OUTPUT 4 -d {ip} -p tcp -j ACCEPT -m comment --comment "call-id:{call_id}"
```
Use `subprocess.run(..., check=True, capture_output=True, text=True)`. If it fails (e.g., comment module unavailable), fall back to:
```
iptables -I OUTPUT 4 -d {ip} -p tcp -j ACCEPT
```
Log whether comment-based or fallback rule was used.

**remove_iptables_rule(ip, call_id) function:** First try with comment matching:
```
iptables -D OUTPUT -d {ip} -p tcp -j ACCEPT -m comment --comment "call-id:{call_id}"
```
If that fails, try without comment:
```
iptables -D OUTPUT -d {ip} -p tcp -j ACCEPT
```
Use `check=False` -- don't fail if rule already removed.

**setup_default_iptables() function (called at startup):**
1. `iptables -F OUTPUT` -- flush OUTPUT chain
2. `iptables -A OUTPUT -o lo -j ACCEPT` -- allow loopback
3. `iptables -A OUTPUT -m state --state ESTABLISHED,RELATED -j ACCEPT` -- allow response traffic
4. Resolve "proxy" hostname to IP via Docker DNS (try `socket.getaddrinfo("proxy", 8080)` -- this should work via Docker embedded DNS at 127.0.0.11 even with dns:127.0.0.1). If resolution fails, log error but continue (proxy may not be resolvable from shared namespace -- will need to be tested).
5. If proxy IP resolved: `iptables -A OUTPUT -d {proxy_ip} -p tcp --dport 8080 -j ACCEPT`
6. `iptables -P OUTPUT DROP` -- default policy DROP

**POST /register endpoint:**
Request body JSON: `{"call_id": "uuid", "domain": "example.com"}`
1. Parse JSON body
2. Resolve domain to IP via `resolve_domain(domain)`
3. If resolution fails, return 400 `{"status": "error", "reason": "DNS resolution failed for {domain}"}`
4. Calculate `expires_at = datetime.utcnow() + timedelta(seconds=CALL_TTL_SECONDS)`
5. INSERT into calls table: call_id, domain, ip_address, created_at (now), expires_at, used=0
6. Add iptables ACCEPT rule for the resolved IP
7. Return 200 `{"status": "ok", "ip": "{ip}", "expires_at": "{expires_at}"}`
8. On any error (duplicate call_id, DB error), return 400/500 with error reason

**GET /validate?call_id=X endpoint:**
1. Parse call_id from query string
2. SELECT from calls WHERE call_id = ? AND used = 0 AND expires_at > datetime('now')
3. If found: UPDATE calls SET used = 1 WHERE call_id = ?; return 200 `{"valid": true, "domain": "{domain}"}`
4. If not found: return 200 `{"valid": false}`

**GET /health endpoint:** Return 200 `"OK"` (same as current stub).

**cleanup_expired() function (background thread):**
1. SELECT call_id, ip_address FROM calls WHERE expires_at <= datetime('now') AND used = 0
2. For each expired entry: remove iptables rule via `remove_iptables_rule(ip, call_id)`
3. DELETE FROM calls WHERE expires_at <= datetime('now')
4. Run every CLEANUP_INTERVAL_SECONDS (5 seconds) using `threading.Timer` in a loop

**Startup sequence (in `if __name__ == '__main__':`):**
1. Configure logging: `logging.basicConfig(level=logging.INFO, format='%(asctime)s %(levelname)s %(message)s')`
2. Call `init_db()`
3. Call `setup_default_iptables()` -- wrap in try/except, log if iptables commands fail (may fail outside Docker)
4. Start cleanup thread (daemon=True so it dies with the main process)
5. Start HTTP server on 0.0.0.0:8088

**Logging:** Use Python logging module. Log all register/validate requests, iptables rule add/remove, cleanup sweeps, and errors. Use `log_message` override on the handler to use Python logging instead of stderr.
