**ENV-01: Secrets from env_file are available in proxy container**
```bash
# Run: docker compose exec -T proxy printenv TEST_SECRET_ALPHA
# Expect: output equals "alpha_secret_value_12345"
result=$(docker compose exec -T proxy printenv TEST_SECRET_ALPHA 2>/dev/null)
test "$result" = "alpha_secret_value_12345"
report "ENV-01" "Secrets from env_file available in proxy" $?
```

**ENV-02: Adding new secret works without editing docker-compose.yml**
This is proven by ENV-01 -- `TEST_SECRET_ALPHA` is not in docker-compose.yml but the proxy has it. The test verifies by checking the var is NOT mentioned in docker-compose.yml AND is present in the proxy:
```bash
# Verify TEST_SECRET_ALPHA is NOT in docker-compose.yml
! grep -q 'TEST_SECRET_ALPHA' docker-compose.yml
dc_check=$?
# AND it IS available in proxy
result=$(docker compose exec -T proxy printenv TEST_SECRET_ALPHA 2>/dev/null)
test "$result" = "alpha_secret_value_12345"
env_check=$?
# Both must pass
test "$dc_check" -eq 0 -a "$env_check" -eq 0
report "ENV-02" "New secret works without docker-compose.yml edit" $?
```

**ENV-03: Claude container does NOT have secret env vars**
```bash
# The claude container should NOT have TEST_SECRET_ALPHA or GITHUB_TOKEN
# (GITHUB_TOKEN was in env_file but not in claude's environment: block)
claude_env=$(docker compose exec -T claude env 2>/dev/null)
! echo "$claude_env" | grep -q 'TEST_SECRET_ALPHA'
alpha_clean=$?
! echo "$claude_env" | grep -q 'GITHUB_TOKEN'
github_clean=$?
test "$alpha_clean" -eq 0 -a "$github_clean" -eq 0
report "ENV-03" "Claude container has no secret env vars" $?
```

**ENV-04: Proxy still redacts secrets with env_file loading**
Use the same mock upstream pattern from test-phase3.sh:
- Start a mock HTTP server inside the proxy container on port 9999 that echoes back what it receives
- Use a docker-compose override to set `REAL_ANTHROPIC_BASE_URL=http://localhost:9999`
- Send a request containing `ghp_test_token_for_phase7` through the proxy
- Verify the mock received `PLACEHOLDER_GITHUB` instead of the real token

If this is too complex for a single task, a simpler alternative: verify the proxy has the GITHUB_TOKEN env var AND the whitelist is readable, and rely on test-phase3.sh for full redaction testing. Use the simpler approach:
```bash
# Verify proxy has GITHUB_TOKEN from env_file
result=$(docker compose exec -T proxy printenv GITHUB_TOKEN 2>/dev/null)
test "$result" = "ghp_test_token_for_phase7"
has_token=$?
# Verify proxy can read whitelist (redaction config)
wl_check=$(docker compose exec -T proxy cat /etc/claude-secure/whitelist.json 2>/dev/null | jq -r '.secrets[0].env_var')
test "$wl_check" = "GITHUB_TOKEN"
has_whitelist=$?
test "$has_token" -eq 0 -a "$has_whitelist" -eq 0
report "ENV-04" "Proxy has secrets and whitelist for redaction" $?
```

**ENV-05: System works with minimal .env (auth only)**
- Stop the current environment
- Create a minimal temp .env with only `ANTHROPIC_API_KEY=sk-ant-test-minimal`
- Export SECRETS_FILE to that minimal file
- Run `docker compose up -d`
- Verify proxy starts and is responsive (health check or simple printenv)
- Verify proxy does NOT have GITHUB_TOKEN or TEST_SECRET_ALPHA
```bash
docker compose down >/dev/null 2>&1
# Create minimal .env
echo "ANTHROPIC_API_KEY=sk-ant-test-minimal" > "$TEMP_ENV_MINIMAL"
export SECRETS_FILE="$TEMP_ENV_MINIMAL"
export ANTHROPIC_API_KEY=sk-ant-test-minimal
docker compose up -d >/dev/null 2>&1
sleep 3
# Proxy should be running
docker compose ps --status running --format '{{.Service}}' | grep -q proxy
running=$?
# Proxy should NOT have GITHUB_TOKEN
! docker compose exec -T proxy printenv GITHUB_TOKEN >/dev/null 2>&1
no_github=$?
test "$running" -eq 0 -a "$no_github" -eq 0
report "ENV-05" "System starts with auth-only .env (no secrets)" $?
```
