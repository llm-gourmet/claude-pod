**docker-compose.yml changes:**

1. Add `env_file:` directive to the `proxy` service, immediately before the `environment:` block:
```yaml
    env_file:
      - ${SECRETS_FILE:-/dev/null}
```
The fallback `/dev/null` ensures docker compose does not fail when SECRETS_FILE is unset (e.g., running docker compose directly without the CLI wrapper).

2. Remove these three lines from the proxy service `environment:` block (they are the hardcoded secret var names that env_file replaces):
```
      - GITHUB_TOKEN=${GITHUB_TOKEN:-}
      - STRIPE_KEY=${STRIPE_KEY:-}
      - OPENAI_API_KEY=${OPENAI_API_KEY:-}
```

3. Keep these lines in the proxy `environment:` block (they are infrastructure config, not user secrets):
```
      - REAL_ANTHROPIC_BASE_URL=https://api.anthropic.com
      - WHITELIST_PATH=/etc/claude-secure/whitelist.json
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY:-}
      - CLAUDE_CODE_OAUTH_TOKEN=${CLAUDE_CODE_OAUTH_TOKEN:-}
      - LOG_ANTHROPIC=${LOG_ANTHROPIC:-0}
```

4. The `claude` service `environment:` block stays EXACTLY as-is. No env_file on claude. The claude container must NOT receive secret values.

5. The `validator` service stays EXACTLY as-is. No env_file on validator.

**bin/claude-secure changes:**

After the existing `export LOG_DIR="$CONFIG_DIR/logs"` line (currently line 29), add:
```bash
# Set SECRETS_FILE for docker compose env_file directive (loads all secrets into proxy container)
export SECRETS_FILE="$CONFIG_DIR/.env"
```

This must come after the `source "$CONFIG_DIR/.env"` block and before the `export COMPOSE_FILE` line.
