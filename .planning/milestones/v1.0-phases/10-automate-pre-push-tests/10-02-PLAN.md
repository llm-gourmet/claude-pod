Replace the entire `git-hooks/pre-push` with a new implementation. The hook must implement all 9 locked decisions (D-01 through D-09). Here is the complete specification:

**Shebang and setup:**
```bash
#!/bin/bash
set -uo pipefail
```

**Constants:**
```bash
REPO_ROOT="$(git rev-parse --show-toplevel)"
TEST_MAP="$REPO_ROOT/tests/test-map.json"
TEST_DIR="$REPO_ROOT/tests"
TEST_ENV="$REPO_ROOT/tests/test.env"
RESULTS_FILE=$(mktemp)
TEST_WHITELIST=$(mktemp)
cp "$REPO_ROOT/config/whitelist.json" "$TEST_WHITELIST"
trap 'rm -f "$RESULTS_FILE" "$TEST_WHITELIST"' EXIT
```

**Section 1: Read refs from stdin (Pitfall 1 from research)**
Read stdin per git pre-push protocol. Each line has `local_ref local_sha remote_ref remote_sha`. Accumulate changed files using `git diff --name-only`. Handle new branches (remote_sha is all zeros) by falling back to `git merge-base HEAD main`. Handle empty stdin (no-op push) by exiting 0. Deduplicate with `sort -u`.

**Section 2: RUN_ALL_TESTS override (D-02)**
If `RUN_ALL_TESTS=1` is set, set `SELECTED_TESTS` to all `test-phase*.sh` files in `$TEST_DIR` and skip to Section 4.

**Section 3: Smart test selection (D-01, D-03)**
For each changed file, check against `always_skip` patterns in test-map.json first. A file matches `always_skip` if it starts with the skip pattern (for directory prefixes like `.planning/`) or matches the glob (for `*.md`). If ALL changed files match `always_skip`, print "No testable changes detected. Skipping tests." and exit 0 (D-03).

For files that are not skipped, match against `mappings[].paths` in test-map.json. A changed file matches a mapping path if the file starts with that path string (prefix matching). Collect all unique test script names from matched mappings.

Use jq to do the matching. The algorithm:
1. Read all changed files
2. For each file, check if it matches any `always_skip` pattern
3. For non-skipped files, find matching `mappings[].tests`
4. Deduplicate test names
5. If no tests matched (but some files were not skipped), run ALL tests as a safety fallback

**Section 4: Test instance setup (D-04, D-09)**
Export environment variables for the dedicated test instance. IMPORTANT: Use the temp copy of whitelist.json (stored in `$TEST_WHITELIST`) instead of the repo file directly. Research doc Pitfall 4 warns that test-phase3.sh modifies whitelist.json during tests -- using a temp copy prevents the working tree from being dirtied by an interrupted hook run.
```bash
export COMPOSE_PROJECT_NAME="claude-test"
export COMPOSE_FILE="$REPO_ROOT/docker-compose.yml"
export SECRETS_FILE="$TEST_ENV"
export WHITELIST_PATH="$TEST_WHITELIST"
export WORKSPACE_PATH="${TMPDIR:-/tmp}/claude-test-workspace"
export LOG_DIR="${TMPDIR:-/tmp}/claude-test-logs"
```
Create `$WORKSPACE_PATH` and `$LOG_DIR` directories if they do not exist (`mkdir -p`).

**Section 5: Print header**
```
========================================
  Pre-push: Running integration tests
========================================
Instance: claude-test (isolated)
Suites:   N selected
```

**Section 6: Run test suites with clean state (D-08)**
For each test script in SELECTED_TESTS:
1. `docker compose down --volumes --remove-orphans --timeout 5 2>/dev/null` (clean slate)
2. `docker compose up -d --wait --timeout 60` (start fresh). If this fails, print "FATAL: containers failed to start for $test_name" and break.
3. Run `bash "$TEST_DIR/$test_script" 2>&1` capturing output to a variable
4. Record exit code
5. Echo the output (so user sees live progress)
6. Extract PASS/FAIL lines from output: `grep -E '(PASS|FAIL)$'` and append to `$RESULTS_FILE` with suite name prefix
7. Track per-suite pass/fail status

**Section 7: Teardown logic (D-05)**
If all suites passed:
- `docker compose down --volumes --remove-orphans --timeout 5 2>/dev/null`
- Print "All tests passed. Containers torn down. Pushing..."
- Exit 0

If any suite failed:
- Do NOT run `docker compose down` (leave containers up for debugging)
- Print summary table (Section 8)
- Print "Containers left running for debugging: docker compose -p claude-test exec claude bash"
- Print "Tear down manually: docker compose -p claude-test down --volumes"
- Exit 1

**Section 8: Summary table on failure (D-07)**
Parse `$RESULTS_FILE` to build a summary table. The file contains lines like:
```
  DOCK-01    Description text                                             PASS
  SECR-04    Some test                                                    FAIL
```

Print:
```
========================================
  Pre-Push Test Results
========================================

Suite              Status
-----------        ------
test-phase1        PASS
test-phase3        FAIL

  Failed requirements:
```
Then grep FAIL lines from RESULTS_FILE and print them.

```
========================================
Push blocked: N suite(s) failed.
========================================
```

**Error handling:**
- If `jq` is not installed, print error and exit 1
- If `docker` is not available, print error and exit 1
- If `test-map.json` does not exist, fall back to running all tests
- Use `set -uo pipefail` but handle individual command failures gracefully (do not exit on first docker compose failure)
