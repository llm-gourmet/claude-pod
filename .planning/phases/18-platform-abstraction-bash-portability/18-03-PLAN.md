<objective>
Patch every host-side script that runs as a fresh shell invocation with the standard prologue: bash 4+ re-exec guard (before any bash 4+ syntax can be parsed), `source lib/platform.sh`, then `claude_secure_bootstrap_path`. After this plan, on macOS plain `date`, `stat`, `readlink`, `realpath`, `sed`, `grep` resolve to GNU implementations from `$(brew --prefix)/opt/coreutils/libexec/gnubin`, and the script body is free to use bash 4+ idioms because Apple's bash 3.2 has been replaced via re-exec.

Purpose: PORT-01 and PORT-02 are the silent-failure surface. Every existing `realpath -m`, `date -d`, `[[ ... ]]`, `[ ${#missing[@]} -gt 0 ]` in bin/claude-secure works perfectly on Linux but breaks on macOS without the prologue. By funneling through one library + one prologue, we get a single audit point and a one-line fix per script.
