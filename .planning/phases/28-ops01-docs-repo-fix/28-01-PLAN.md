<objective>
Fix the OPS-01 forward-compatibility defect in `bin/claude-secure` `do_spawn` that silently clobbers `REPORT_REPO` / `REPORT_BRANCH` for profiles migrated to the Phase 23 canonical `docs_repo` schema. The bug is three `jq` reads at lines 2077-2079 that unconditionally re-read the legacy-named profile fields (`.report_repo`, `.report_branch`) and export empty strings, overwriting the correct values that `resolve_docs_alias` back-filled earlier in the spawn lifecycle. The result is silent report skip via `publish_report` `return 2`, masking a broken publishing pipeline for any user who adopts the Phase 23 schema.

This plan lands a failing regression test FIRST (Wave 0 / RED gate) so the defect is captured, then patches the three `jq` expressions (GREEN). Both tasks ship in a single plan because the fix is surgical (three expressions on three consecutive lines) and must be sequenced test-before-code to prove the test actually covers the bug.

Purpose: Close the OPS-01 Contract 5 partial gap identified in `.planning/v2.0-MILESTONE-AUDIT.md` so profiles migrated to Phase 23 never silently skip report publishing.
Output: Updated `tests/test-phase16.sh` (new test + dispatch registration) and patched `bin/claude-secure` (three jq fallback chains).
</objective>
