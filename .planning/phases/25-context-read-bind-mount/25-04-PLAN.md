## Edit 1: `_spawn_ctx_background` — replace `sleep 2` with a container-ready poll loop

Current body (lines ~326-343) ends with:

```bash
  CLAUDE_SECURE_FAKE_CLAUDE_STDOUT=/dev/null \
    CLAUDE_SECURE_FAKE_CLAUDE_EXIT=0 \
    "$PROJECT_DIR/bin/claude-secure" --profile "$dest" spawn \
      --event '{"event_type":"manual","repository":{"full_name":"owner/test"}}' \
      >/dev/null 2>&1 &
  SPAWN_PID=$!
  sleep 2
  source_cs
  SPAWN_PROJECT=$(spawn_project_name "$dest" 2>/dev/null || echo "cs-${dest}")
}
```

Rewrite the tail so that AFTER `SPAWN_PID=$!` and BEFORE the caller invokes `docker compose exec`, we poll until the claude container is actually running. Required shape:

1. `source_cs` first so `spawn_project_name` is defined.
2. Compute `SPAWN_PROJECT` as before.
3. Poll loop — up to 30 attempts at 0.5s each (~15s total budget). On each iteration:
   - `docker compose -p "$SPAWN_PROJECT" ps --status=running --services 2>/dev/null | grep -Fxq claude` — if true, container is up; break.
   - Otherwise: if the background spawn process has already died (`! kill -0 "$SPAWN_PID" 2>/dev/null`), break — the caller will see exec fail and return 1, which is correct.
   - `sleep 0.5`.
4. Leave `SPAWN_PID` / `SPAWN_PROJECT` exported regardless of how the loop exited — `_kill_spawn` is idempotent and always runs on the caller's failure path.
5. Do NOT `return` a non-zero code from the poll loop; the caller's existing `|| { _kill_spawn; return 1; }` chains already handle the failure case via their own exec-level check. The poll loop's job is to WAIT, not to decide.

Reference skeleton (adapt to match surrounding style — this is illustrative, not verbatim):

```bash
  SPAWN_PID=$!
  # Wait for the claude container to actually enter the running state.
  # Fixed sleeps race on WSL2 where docker compose up -d --wait can exceed 2s.
  source_cs
  SPAWN_PROJECT=$(spawn_project_name "$dest" 2>/dev/null || echo "cs-${dest}")
  local _i
  for _i in $(seq 1 30); do
    if docker compose -p "$SPAWN_PROJECT" ps --status=running --services 2>/dev/null \
         | grep -Fxq claude; then
      break
    fi
    if ! kill -0 "$SPAWN_PID" 2>/dev/null; then
      break
    fi
    sleep 0.5
  done
}
```

Notes:
- `--services` restricts output to service names (`claude`, `proxy`, `validator`), one per line. `grep -Fxq claude` matches the exact service name and avoids substring false positives.
- Do NOT use `docker ps` directly — we want compose-project-scoped visibility, not global.
- Do NOT use `docker compose ... --status=healthy` — there is no healthcheck on the claude service; healthy will never fire.
- Budget ~15s is ample for WSL2 cold start while staying well under the test-suite patience threshold.

## Edit 2: `test_agent_docs_no_git_dir_in_container` — add exec-health guard before the `.git` assertion

Current body (lines ~381-391):

```bash
test_agent_docs_no_git_dir_in_container() {
  _docker_gate_or_skip || return 0
  _spawn_ctx_background "ctx-nogit-test" || { _kill_spawn; return 1; }
  docker compose -p "$SPAWN_PROJECT" exec -T claude ls /agent-docs/.git >/dev/null 2>&1 && {
    _kill_spawn
    echo "FAIL: /agent-docs/.git exists inside container" >&2
    return 1
  }
  _kill_spawn
  return 0
}
```

The bug: if `docker compose exec` fails because the container is not up, `ls /agent-docs/.git` also fails, the `&&` branch is skipped, and `return 0` fires — the test passes without ever verifying anything. Once Edit 1 lands this is less likely to fire, but the guard is still required so the failure mode is LOUD rather than silent-false-positive.

Insert an exec-reachability check that runs a trivial no-op command inside the container. If that fails, the container is not reachable and the test must FAIL (not pass).

Rewrite shape:

```bash
test_agent_docs_no_git_dir_in_container() {
  _docker_gate_or_skip || return 0
  _spawn_ctx_background "ctx-nogit-test" || { _kill_spawn; return 1; }
  # Guard: if the container is not actually reachable, fail loudly instead of
  # silently passing because both exec calls return non-zero.
  if ! docker compose -p "$SPAWN_PROJECT" exec -T claude true >/dev/null 2>&1; then
    _kill_spawn
    echo "FAIL: claude container not reachable via docker compose exec" >&2
    return 1
  fi
  docker compose -p "$SPAWN_PROJECT" exec -T claude ls /agent-docs/.git >/dev/null 2>&1 && {
    _kill_spawn
    echo "FAIL: /agent-docs/.git exists inside container" >&2
    return 1
  }
  _kill_spawn
  return 0
}
```

The guard uses `true` (POSIX shell builtin / GNU coreutils — both present in the claude image) and discards its output. It proves the container exists AND exec is functional AND the working shell inside resolves PATH. It is the cheapest possible liveness probe.
