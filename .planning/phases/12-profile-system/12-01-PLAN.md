    `validate_profile_name()` (per D-01, reuse from validate_instance_name):
    ```bash
    validate_profile_name() {
      local name="$1"
      if [[ ! "$name" =~ ^[a-z0-9][a-z0-9-]*$ ]]; then
        echo "ERROR: Profile name must be DNS-safe (lowercase alphanumeric and hyphens, must start with letter or digit)" >&2
        return 1
      fi
      if [ ${#name} -gt 63 ]; then
        echo "ERROR: Profile name must be 63 characters or less" >&2
        return 1
      fi
    }
    ```
    IMPORTANT: Use `return 1` not `exit 1` so tests can source the function.

    `validate_profile()` (PROF-03, fail-closed, per 12-RESEARCH.md Pattern 2):
    - Check profile dir exists at $CONFIG_DIR/profiles/$name/
    - Check profile.json exists
    - Check profile.json is valid JSON (jq empty)
    - Check profile.json has workspace field (jq -r '.workspace // empty')
    - Check workspace path is a directory
    - Check .env exists
    - Check whitelist.json exists and is valid JSON
    - Each failure: echo "ERROR: Profile '$name' ..." to stderr, return 1
    - NEVER fall back to a default profile

    `create_profile()` (PROF-01, per D-07, D-08, D-09, D-12):
    - Create in tmpdir first, mv atomically (per Pitfall 6 in research)
    - Prompt for workspace path (default: $HOME/claude-workspace-$name)
    - realpath -m and mkdir -p the workspace
    - Build profile.json with jq: `jq -n --arg ws "$ws_path" '{"workspace": $ws}'`
    - Copy whitelist template from $APP_DIR/config/whitelist.json
    - Auth setup: check for existing profile .env to copy (same pattern as create_instance), else prompt interactively
    - Atomic mv $tmpdir to $CONFIG_DIR/profiles/$name
    - chmod 600 on .env

    `setup_profile_auth()` (reuse logic from setup_instance_auth):
    - Same OAuth/API key choice prompt
    - Write to $1/.env (tmpdir path during creation)
    - Append comments about secrets
    - chmod 600

    `resolve_profile_by_repo()` (PROF-02, per D-04, D-05, D-06):
    ```bash
    resolve_profile_by_repo() {
      local target_repo="$1"
      shopt -s nullglob
      for pdir in "$CONFIG_DIR/profiles"/*/; do
        [ -f "$pdir/profile.json" ] || continue
        local repo
        repo=$(jq -r '.repo // empty' "$pdir/profile.json")
        if [ "$repo" = "$target_repo" ]; then
          basename "$pdir"
          shopt -u nullglob
          return 0
        fi
      done
      shopt -u nullglob
      echo "ERROR: No profile found for repo '$target_repo'" >&2
      return 1
    }
    ```

    `merge_whitelists()` (D-16, superuser mode):
    - Iterate all profile dirs, merge secrets arrays and readonly_domains arrays
    - Use jq --slurpfile with unique_by(.env_var) for secrets, unique for domains
    - Handle zero profiles: return empty whitelist (per research Open Question 1 -- allow with warning)
    - Write merged JSON to stdout

    `merge_env_files()` (D-16, superuser mode):
    - Concatenate all profile .env files to a temp file
    - Ensure newline separator between files
    - Echo temp file path to stdout

    `load_profile_config()` (sets Docker Compose env vars):
    ```bash
    load_profile_config() {
      local name="$1"
      local pdir="$CONFIG_DIR/profiles/$name"
      WORKSPACE_PATH=$(jq -r '.workspace' "$pdir/profile.json")
      export COMPOSE_PROJECT_NAME="claude-${name}"
      export COMPOSE_FILE="$APP_DIR/docker-compose.yml"
      export WORKSPACE_PATH
      export SECRETS_FILE="$pdir/.env"
      export WHITELIST_PATH="$pdir/whitelist.json"
      export LOG_DIR="$CONFIG_DIR/logs"
      export LOG_PREFIX="${name}-"
      set -a
      source "$pdir/.env"
      set +a
    }
    ```

    `load_superuser_config()` (D-15, D-16, D-17):
    - If no DEFAULT_WORKSPACE in config.sh, prompt and save it
    - Call merge_whitelists, write to temp file
    - Call merge_env_files, get temp file path
    - Export COMPOSE_PROJECT_NAME="claude-superuser"
    - Export merged paths as SECRETS_FILE, WHITELIST_PATH
    - Export DEFAULT_WORKSPACE as WORKSPACE_PATH
    - Register temp files for cleanup

    `list_profiles()` (D-14):
    - printf header: PROFILE, REPO, STATUS, WORKSPACE (per D-14 format from research)
    - shopt -s nullglob for profile iteration
    - For each profile dir: read repo and workspace from profile.json via jq, check docker compose status
    - Handle no profiles: "No profiles configured."

    `cleanup()`:

    - Called via `trap cleanup EXIT`

    - If PROFILE is empty (superuser mode): load_superuser_config
