Create `install.sh` in the project root with `#!/bin/bash` and `set -euo pipefail`.

Structure with a `main()` function calling these functions in order:

1. **Logging helpers** (per Claude's discretion on colors):
   - `log_info()`, `log_warn()`, `log_error()` with colored prefixes (GREEN [INFO], YELLOW [WARN], RED [ERROR])
   - `NC='\033[0m'` for reset

2. **`check_dependencies()`** (per D-01, D-02 -- INST-01):
   - Use `command -v` to check: `docker`, `curl`, `jq`, `uuidgen`
   - Collect ALL missing into a `missing=()` array before reporting
   - Install hints: `docker` -> "https://docs.docker.com/engine/install/", `curl` -> "apt install curl", `jq` -> "apt install jq", `uuidgen` -> "apt install uuid-runtime"
   - Docker Compose v2: if docker exists, run `docker compose version >/dev/null 2>&1`. If fails, check if `docker-compose` (v1) exists -- if so, error "you have v1 which is deprecated -- upgrade Docker". Otherwise "install Docker Compose plugin".
   - If `missing` array is non-empty, print all missing with hints and `exit 1`

3. **`detect_platform()`** (per D-14, D-15, D-16 -- INST-02, PLAT-01, PLAT-02, PLAT-03):
   - Check `grep -qi microsoft /proc/version 2>/dev/null` -- if true, `PLATFORM="wsl2"`, else `PLATFORM="linux"`
   - If WSL2: run `docker info --format '{{.OperatingSystem}}'` and check for "docker desktop" (case-insensitive). If found, `log_warn "Docker Desktop detected. iptables may not work correctly. Recommended: use Docker CE installed directly in WSL2."`
   - Log iptables version: `iptables -V 2>/dev/null || echo "not found"` -- just informational logging
   - `log_info "Detected platform: $PLATFORM"`

4. **`check_existing()`** (idempotency -- Pitfall 7 from research):
   - If `$CONFIG_DIR` (`$HOME/.claude-secure`) exists, `log_warn "Existing installation found at $CONFIG_DIR"` and prompt `read -rp "Overwrite existing installation? [y/N]: " overwrite`
   - If not y/Y, `log_info "Installation cancelled."` and `exit 0`
   - If overwriting and `.env` exists, backup to `.env.backup.$(date +%s)`

5. **`setup_directories()`** (per D-09 -- INST-04):
   - `mkdir -p "$CONFIG_DIR"` and `chmod 700 "$CONFIG_DIR"`

6. **`setup_auth()`** (per D-03, D-04, D-05 -- INST-03):
   - Check `${ANTHROPIC_API_KEY:-}` -- if non-empty, write `ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY` to `$CONFIG_DIR/.env`, log "Using ANTHROPIC_API_KEY from environment", return
   - Check `${CLAUDE_CODE_OAUTH_TOKEN:-}` -- if non-empty, write to `.env`, return
   - Interactive prompt: print "Choose authentication method:", "  1) OAuth token [recommended]", "  2) API key", then `read -rp "Choice [1]: " auth_choice` with default "1"
   - If choice 1: prompt "Run 'claude setup-token' first to get your token.", then `read -rsp "OAuth token: " token`, write `CLAUDE_CODE_OAUTH_TOKEN=$token` to `.env`
   - If choice 2: `read -rsp "API key: " key`, write `ANTHROPIC_API_KEY=$key` to `.env`
   - `chmod 600 "$CONFIG_DIR/.env"`

7. **`setup_workspace()`** (per D-10 -- INST-04):
   - `read -rp "Workspace path [$HOME/claude-workspace]: " ws_path`
   - Default to `$HOME/claude-workspace` if empty
   - Resolve to absolute path: `ws_path="$(cd "$(dirname "$ws_path")" 2>/dev/null && pwd)/$(basename "$ws_path")"` -- if dirname fails (doesn't exist yet), use the path as-is after expanding `~`
   - Simpler: use `realpath -m "$ws_path"` (creates missing parent logic, -m means no requirement to exist)
   - `mkdir -p "$ws_path"`
   - Write to `$CONFIG_DIR/config.sh`:
     ```
     WORKSPACE_PATH="$ws_path"
     PLATFORM="$PLATFORM"
     APP_DIR="$app_dir"
     ```
   - `log_info "Workspace: $ws_path"`

8. **`copy_app_files()`** (per D-11 -- INST-04):
   - `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` at top of script
   - `app_dir="$CONFIG_DIR/app"`
   - If `SCRIPT_DIR` equals `app_dir`, log "Running from installed location, skipping copy"
   - Else if `$SCRIPT_DIR/docker-compose.yml` exists, `rm -rf "$app_dir" 2>/dev/null; cp -r "$SCRIPT_DIR" "$app_dir"`, log "Copied project files to $app_dir"
   - Else error "Cannot find docker-compose.yml in $SCRIPT_DIR. Run from the claude-secure project directory." and exit 1
   - Copy default whitelist if `$CONFIG_DIR/whitelist.json` doesn't exist: `cp "$app_dir/config/whitelist.json" "$CONFIG_DIR/whitelist.json"`
   - Create symlink: `ln -sf "$CONFIG_DIR/whitelist.json" "$app_dir/config/whitelist.json"` so docker-compose.yml's relative mount reads the user's copy

9. **`build_images()`** (per D-12, D-13 -- INST-05):
   - `log_info "Building Docker images..."`
   - `cd "$app_dir" && docker compose build` (must cd because compose file uses relative paths for build contexts)
   - Verify: `docker compose config --quiet` -- if fails, error and exit
   - `log_info "Docker images built successfully"`

10. **`install_cli()`** (per D-06 -- INST-06):
    - `cli_src="$CONFIG_DIR/app/bin/claude-secure"`
    - Try `/usr/local/bin/claude-secure`: if `/usr/local/bin` is writable, `cp "$cli_src" "/usr/local/bin/claude-secure" && chmod 755`
    - Else if `sudo` available, `sudo cp "$cli_src" "/usr/local/bin/claude-secure" && sudo chmod 755`
    - Else fallback `$HOME/.local/bin/claude-secure`: `mkdir -p "$HOME/.local/bin"`, copy, chmod 755
    - If fallback used and `$HOME/.local/bin` not in PATH, `log_warn "$HOME/.local/bin is not in PATH. Add it to your shell profile."`

11. **`main()`**: Banner "=== claude-secure installer ===" then call all functions in order, finish with "Installation complete! Run 'claude-secure' to start."

Variables at top of script: `CONFIG_DIR="$HOME/.claude-secure"`, `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`, `PLATFORM=""`, `app_dir=""`.

12. **Source guard** (for testability -- allows `source install.sh` without executing main):
    - At the bottom of the script, wrap the `main "$@"` invocation:
    ```bash
    if [[ "${BASH_SOURCE[0]}" == "${0}" ]]; then
      main "$@"
    fi
    ```
    - This enables Plan 04-02's test script to source individual functions without triggering the full installer.

Make executable: the file should have a shebang but the Write tool can't set permissions -- add a note in verify.

   ```bash
   case "${1:-}" in

       docker compose down

       ;;
     *)
       docker compose up -d
       docker compose exec -it claude claude
       ;;
   esac
   ```

4. The default command (`claude-secure` no args) runs `docker compose up -d` then `docker compose exec -it claude claude` per D-07.
