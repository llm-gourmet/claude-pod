<!-- Stop hook stdin JSON schema (from 26-RESEARCH.md, verified via code.claude.com/docs/en/hooks) -->
<!-- Fields the hook reads: stop_hook_active (bool, defaults false) -->

Stop hook input (stdin, JSON):
```json
{
  "session_id": "...",
  "hook_event_name": "Stop",
  "stop_hook_active": false,
  ...
}
```

Stop hook output modes:
1. Yield silently: exit 0 with no stdout
2. Block with reprompt: stdout=`{"decision":"block","reason":"..."}`, exit 0
3. On parse error: default to "treat as stop_hook_active=false" and proceed with normal spool check (Pitfall 6)

Testability contract (from Plan 01 test harness):
- TEST_SPOOL_FILE_OVERRIDE: if set, use this path instead of /var/log/claude-secure/spool.md as SPOOL_FILE

settings.json structure (from 26-RESEARCH.md — exact form):
```json
{
  "hooks": {
    "PreToolUse": [ ... existing ... ],
    "Stop": [
      { "hooks": [ { "type": "command", "command": "/etc/claude-secure/hooks/stop-hook.sh" } ] }
    ]
  }
}
```
Note: Stop hooks do NOT support matchers — omit the matcher field from the Stop entry.
</interfaces>
