**Test MULTI-01: --instance flag is required**
Verify that running `bin/claude-secure` without `--instance` exits non-zero with error message. Use a temp CONFIG_DIR with a dummy config.sh so the installation check passes. Set HOME to temp dir. Verify that the error output contains "instance NAME is required".

**Test MULTI-02: DNS-safe instance name validation**
Source the `validate_instance_name` function from bin/claude-secure (or replicate it). Test that:
- `myproject` is valid (exits 0)
- `my-project` is valid
- `project123` is valid
- `MyProject` is invalid (uppercase)
- `my_project` is invalid (underscore)
- `my project` is invalid (space)
- `-project` is invalid (starts with hyphen)

Since sourcing the full CLI will trigger side effects, extract the validation regex and test it directly:
```bash
test_dns_name() {
  local name="$1"
  [[ "$name" =~ ^[a-z0-9][a-z0-9-]*$ ]]
}
```

**Test MULTI-03: Migration from single-instance**
Create a temp CONFIG_DIR with old layout: config.sh (with WORKSPACE_PATH, PLATFORM, APP_DIR), .env (with auth). Run the migration logic. Verify:
- `instances/default/` directory exists
- `instances/default/.env` exists and contains auth
- `instances/default/config.sh` contains WORKSPACE_PATH
- Global config.sh does NOT contain WORKSPACE_PATH
- Global config.sh contains APP_DIR and PLATFORM
- Root-level .env no longer exists

**Test MULTI-04: COMPOSE_PROJECT_NAME isolation**
Verify docker-compose.yml has no `container_name` directives. Verify that running `COMPOSE_PROJECT_NAME=claude-test1 docker compose -f docker-compose.yml config` produces container names containing "test1". Same for "test2" -- names must differ.

**Test MULTI-05: Per-instance config files**
Create two instance directories in temp CONFIG_DIR. Verify each has its own config.sh, .env, whitelist.json. Verify files are independent (different content).

**Test MULTI-06: LOG_PREFIX in docker-compose.yml and service code**
Verify docker-compose.yml contains `LOG_PREFIX=${LOG_PREFIX:-}` for all three services. Verify proxy.js contains `LOG_PREFIX`. Verify validator.py contains `LOG_PREFIX` or `log_prefix`. Verify pre-tool-use.sh contains `LOG_PREFIX`.

**Test MULTI-07: list command output format**
Create temp CONFIG_DIR with instances/foo/ and instances/bar/ (each with config.sh containing WORKSPACE_PATH). Run the list logic and verify output contains both instance names.

**Test MULTI-08: Instance auto-creation directory structure**
After create_instance runs (or simulated), verify:
- `instances/{name}/config.sh` exists
- `instances/{name}/.env` exists
- `instances/{name}/whitelist.json` exists
- whitelist.json matches the template from config/whitelist.json

**Test MULTI-09: Global config.sh contains only APP_DIR and PLATFORM**
Run install.sh in a test mode (source the functions). Verify the global config.sh written by the installer contains APP_DIR and PLATFORM but NOT WORKSPACE_PATH.
