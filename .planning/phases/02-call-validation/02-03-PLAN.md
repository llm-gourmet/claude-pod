**Test CALL-01: Hook intercepts Bash tool calls**
Feed a simple Bash tool call JSON to the hook and verify it processes it (doesn't crash):
```bash
RESULT=$(echo '{"tool_name":"Bash","tool_input":{"command":"echo hello"}}' | \
  docker exec -i claude-secure /etc/claude-secure/hooks/pre-tool-use.sh 2>&1)
```

**Test CALL-06: Validator stores call-IDs with single-use enforcement**
```bash
# Register a call-ID directly with the validator
CALL_ID=$(uuidgen)
RESPONSE=$(docker exec claude-secure curl -s -X POST http://127.0.0.1:8088/register \
  -H "Content-Type: application/json" \
  -d "{\"call_id\":\"${CALL_ID}\",\"domain\":\"api.github.com\"}")
echo "$RESPONSE" | grep -q '"ok"'

# First validate -- should succeed
VAL1=$(docker exec claude-secure curl -s "http://127.0.0.1:8088/validate?call_id=${CALL_ID}")
echo "$VAL1" | jq -r '.valid' | grep -q 'true'

# Second validate -- should fail (single-use)
VAL2=$(docker exec claude-secure curl -s "http://127.0.0.1:8088/validate?call_id=${CALL_ID}")
echo "$VAL2" | jq -r '.valid' | grep -q 'false'
```

**Test CALL-06b: Validator rejects expired call-IDs**
```bash
# Register a call-ID, wait for expiry (>10s), then validate
CALL_ID=$(uuidgen)
docker exec claude-secure curl -s -X POST http://127.0.0.1:8088/register \
  -H "Content-Type: application/json" \
  -d "{\"call_id\":\"${CALL_ID}\",\"domain\":\"api.github.com\"}" > /dev/null
# Wait for expiry + cleanup
sleep 12
VAL=$(docker exec claude-secure curl -s "http://127.0.0.1:8088/validate?call_id=${CALL_ID}")
echo "$VAL" | jq -r '.valid' | grep -q 'false'
```

**Test CALL-07: iptables blocks outbound without call-ID**
```bash
# Attempt direct outbound connection from claude container (should be blocked by iptables DROP)
# Try to connect to a well-known IP (8.8.8.8:443) -- should fail/timeout
RESULT=$(docker exec claude-secure timeout 3 bash -c 'echo | curl -s --connect-timeout 2 https://8.8.8.8 2>&1' || true)
# Should fail (connection refused/timeout due to iptables DROP)
echo "$RESULT" | grep -qE 'refused|timed out|Connection timed out|Could not resolve|Failed to connect|curl.*error' || [ -z "$RESULT" ]
```

**Test CALL-07b: iptables allows traffic to proxy**
```bash
# Connection to proxy on port 8080 should work (allowed by iptables rules)
RESULT=$(docker exec claude-secure curl -s --connect-timeout 3 http://proxy:8080/health 2>&1 || docker exec claude-secure curl -s --connect-timeout 3 http://127.0.0.1:8080/ 2>&1 || true)
# Should get some response (even error page from proxy is fine -- means connection worked)
# The proxy stub returns something on any path
[ -n "$RESULT" ]
```
Note: Since proxy and validator share namespace, test proxy reachability by its Docker IP (may need adjustment based on actual network setup). If proxy is on a different network and the validator shares claude's namespace, the proxy is still reachable via the internal network IP. The iptables rule allows traffic to proxy_ip:8080.
