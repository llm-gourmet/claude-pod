    ### Helper 2: `extract_payload_field`

    Per D-17, D-18, and 15-RESEARCH.md Pattern 6. Takes three args: `event_json` (raw JSON string), `jq_path` (jq filter expression), `default_value` (fallback if jq returns empty). Runs jq, pipes through a python3 heredoc for hygiene (strip NUL + ASCII control chars except \n\r\t; truncate to 8192 bytes UTF-8-safe; append `... [truncated N more bytes]` suffix). Writes to a temp file and appends the temp file path to the existing `_CLEANUP_FILES` bash array. Echoes the temp file path.

    **Hard constraint — DO NOT use `head -c`** (or `cut -b`, or any byte-based bash tool). The negative grep in verify enforces this. UTF-8 truncation MUST go through python3.

    **Hard constraint — stdin, not argv**. Pass the jq output to python3 via stdin (`printf '%s' "$raw" | python3 - "$out_file" <<'PY' ... PY`), never as an argv argument (avoids argv-size limits and shell escaping bugs).

    ```bash
    extract_payload_field() {
      local event_json="$1"
      local jq_path="$2"
      local default_value="${3:-}"

      local raw
      raw=$(echo "$event_json" | jq -r "$jq_path // empty" 2>/dev/null)
      if [ -z "$raw" ]; then
        raw="$default_value"
      fi

      local out_file
      out_file=$(mktemp)
      _CLEANUP_FILES+=("$out_file")

      # Hygiene via python3 (stdin -> argv[1]):
      #   D-18: strip NUL + ASCII control chars except \n \r \t
      #   D-17: UTF-8-safe truncation at 8192 bytes with "... [truncated N more bytes]" suffix
      printf '%s' "$raw" | python3 - "$out_file" <<'PY'
    import sys
    data = sys.stdin.buffer.read()
    keep = {0x09, 0x0A, 0x0D}
    cleaned = bytes(b for b in data if b >= 0x20 or b in keep)
    LIMIT = 8192
    if len(cleaned) > LIMIT:
        truncated = cleaned[:LIMIT].decode("utf-8", errors="ignore").encode("utf-8")
        dropped = len(cleaned) - len(truncated)
        suffix = f"... [truncated {dropped} more bytes]".encode("utf-8")
        cleaned = truncated + suffix
    with open(sys.argv[1], "wb") as f:
        f.write(cleaned)
    PY

      echo "$out_file"
    }
    ```

    _substitute_token_from_file() {
      local rendered="$1"
      local token="$2"
      local value_file="$3"
      echo "$rendered" | awk -v token="{{${token}}}" -v vfile="$value_file" '
        BEGIN {
          value = ""
          while ((getline line < vfile) > 0) {
            if (value == "") value = line
            else value = value "\n" line
          }
          close(vfile)
        }
        {

            n = split($0, parts, token)
            out = parts[1]

            print out
          } else {
            print
          }
        }
      '

    ### Region 2: `render_template()` at line ~400 — replace ENTIRE function

    Per D-16 and 15-RESEARCH.md Pattern 7. Use `extract_payload_field` + `_substitute_token_from_file` for every short variable, and `_substitute_multiline_token_from_file` for ISSUE_BODY and COMMIT_MESSAGE. **NO sed on any payload-derived string** — that is the Pitfall 1 fix.

    **CRITICAL — Gated-substitution rule for variables with multi-event fallbacks (BRANCH, COMMIT_SHA):**

    `_substitute_token_from_file` always replaces the token, even when the value file is empty. That means the naive pattern "substitute from .ref, then re-check if `{{BRANCH}}` is still in the rendered output and fall back" DOES NOT WORK on workflow_run payloads: the first pass replaces `{{BRANCH}}` with the empty string (because `.ref` is absent), and the post-hoc `grep -q '{{BRANCH}}'` check returns false because the token was already consumed. BRANCH silently renders as empty. Same bug for COMMIT_SHA.

    The fix: **read the extracted value from disk and only call the substitutor when it is non-empty.** Only the final successful path (or the final fallback's empty value, after every source is exhausted) writes a substitution.

    Pattern (use this for BRANCH and COMMIT_SHA — both have push-path and workflow_run-path sources):

    ```bash
    # BRANCH: push (.ref with refs/heads/ stripped) -> workflow_run (.workflow_run.head_branch)
    v_file=$(extract_payload_field "$event_json" '.ref | sub("^refs/heads/"; "")' "")
    if [ -s "$v_file" ]; then
      rendered=$(_substitute_token_from_file "$rendered" "BRANCH" "$v_file")
    else
      v_file=$(extract_payload_field "$event_json" '.workflow_run.head_branch' "")
      rendered=$(_substitute_token_from_file "$rendered" "BRANCH" "$v_file")
    fi

    # COMMIT_SHA: push (.after // .head_commit.id) -> workflow_run (.workflow_run.head_sha)
    v_file=$(extract_payload_field "$event_json" '.after // .head_commit.id' "")
    if [ -s "$v_file" ]; then
      rendered=$(_substitute_token_from_file "$rendered" "COMMIT_SHA" "$v_file")
    else
      v_file=$(extract_payload_field "$event_json" '.workflow_run.head_sha' "")
      rendered=$(_substitute_token_from_file "$rendered" "COMMIT_SHA" "$v_file")
    fi
    ```

    The `[ -s "$v_file" ]` test is true only when the file exists AND has non-zero size. Since `extract_payload_field` writes the raw bytes of the jq result (empty string → empty file), this is the correct trigger for "try the next source." Do NOT use `grep -q '{{BRANCH}}' <<<"$rendered"` — that check is unreliable as explained above.

    The same gating pattern MUST be applied to any variable added later that has multiple sources. Single-source variables (REPO_NAME, ISSUE_*, PUSHER, WORKFLOW_*) substitute unconditionally.

    ### Function: `do_replay`

    Place near the existing `do_spawn` function. Key rules:

    1. Parse `REMAINING_ARGS` for a positional `<delivery-substring>`, optional `--profile NAME`, optional `--dry-run`.
    2. If substring empty → print usage and return 1.
    3. If `$CONFIG_DIR/events/` doesn't exist → error.
    4. Glob for `"$events_dir"/*"${delivery_substring}"*.json` with `shopt -s nullglob` to array `matches`.
    5. Zero matches → error, return 1.
    6. Multiple matches → print all candidates to stderr, return 1.
    7. Single match → proceed.
    8. If no explicit profile, parse `repository.full_name` from the event file and call `resolve_profile_by_repo` (existing function in bin/claude-secure at line ~152).
    9. **Exec recursively** — do NOT duplicate do_spawn logic. `exec "$0" --profile "$explicit_profile" spawn --event-file "$event_file"` plus `--dry-run` if set. The `exec` replaces the current shell so no cleanup trap duplication.

    ```bash
    do_replay() {
      local delivery_substring=""
      local explicit_profile="${PROFILE:-}"
      local dry_run=0
      local i=1
      while [ $i -lt ${#REMAINING_ARGS[@]} ]; do
        case "${REMAINING_ARGS[$i]:-}" in
          --profile)  explicit_profile="${REMAINING_ARGS[$((i+1))]:-}"; i=$((i+2)) ;;
          --dry-run)  dry_run=1; i=$((i+1)) ;;
          *)
            if [ -z "$delivery_substring" ]; then
              delivery_substring="${REMAINING_ARGS[$i]}"
            fi
            i=$((i+1))
            ;;
        esac
      done

      if [ -z "$delivery_substring" ]; then
        echo "ERROR: replay requires a delivery-id substring" >&2
        echo "Usage: claude-secure replay <delivery-id> [--profile NAME] [--dry-run]" >&2
        return 1
      fi

      local events_dir="$CONFIG_DIR/events"
      if [ ! -d "$events_dir" ]; then
        echo "ERROR: events directory not found: $events_dir" >&2
        return 1
      fi

      local matches=()
      shopt -s nullglob
      for f in "$events_dir"/*"${delivery_substring}"*.json; do
        matches+=("$f")
      done
      shopt -u nullglob

      if [ ${#matches[@]} -eq 0 ]; then
        echo "ERROR: no event file matching '$delivery_substring' in $events_dir" >&2
        return 1
      fi
      if [ ${#matches[@]} -gt 1 ]; then
        echo "ERROR: multiple event files match '$delivery_substring':" >&2
        for m in "${matches[@]}"; do echo "  $m" >&2; done
        echo "Narrow the substring and retry." >&2
        return 1
      fi

      local event_file="${matches[0]}"

      # D-21: auto-resolve profile from repository.full_name if not explicit
      if [ -z "$explicit_profile" ]; then
        local repo
        repo=$(jq -r '.repository.full_name // empty' "$event_file")
        if [ -z "$repo" ]; then
          echo "ERROR: cannot determine repository from $event_file -- pass --profile" >&2
          return 1
        fi
        explicit_profile=$(resolve_profile_by_repo "$repo") || return 1
      fi

      echo "Replaying $event_file as profile '$explicit_profile'..."

      # D-20 / Pattern 9: exec recursively to avoid duplicating do_spawn logic
      local args=(--profile "$explicit_profile" spawn --event-file "$event_file")
      [ "$dry_run" = 1 ] && args+=(--dry-run)
      exec "$0" "${args[@]}"
    }
    ```
