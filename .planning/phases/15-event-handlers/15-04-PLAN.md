    ### New block (D-12: always refresh)

    ```bash
    # 5b. Copy default prompt templates (D-12: always refresh -- latest templates ship)
    sudo mkdir -p /opt/claude-secure/webhook/templates
    if [ -d "$app_dir/webhook/templates" ]; then
      sudo cp "$app_dir/webhook/templates/"*.md /opt/claude-secure/webhook/templates/
      sudo chmod 644 /opt/claude-secure/webhook/templates/*.md
      log_info "Copied default templates to /opt/claude-secure/webhook/templates/"
    else
      log_warn "Source directory $app_dir/webhook/templates not found -- skipping template copy"
    fi
    ```

    Key behaviors:
    1. `sudo mkdir -p` is idempotent — safe on reinstall.
    2. `sudo cp "$app_dir/webhook/templates/"*.md` uses shell glob expansion — copies ALL .md files without enumeration. The trailing `/*.md` ensures a directory source doesn't accidentally create a nested `templates/templates/` directory.
    3. `sudo chmod 644` sets read-only for non-root (templates are read during spawn, never written).
    4. Fallback branch (log_warn) is defensive — if somehow the templates dir is missing in the source, the installer warns but does not fail. Plan 15-02 ensures the dir exists, so this fallback should never fire in practice.
    5. This block does NOT touch profile-level templates in `~/.claude-secure/profiles/<name>/prompts/` — those remain the user's responsibility (Phase 13 convention).

    ### Anti-patterns to avoid
    - Do NOT use `rsync` — adds a dep not currently required by install.sh.
    - Do NOT `rm -rf /opt/claude-secure/webhook/templates` before copying — `cp` with the same filenames overwrites cleanly, and a rm could race with a running listener reading a template file (unlikely but possible).
    - Do NOT gate the copy on "file already exists" — D-12 says always refresh.
    - Do NOT copy the entire `webhook/` directory recursively — that would unnecessarily recopy listener.py and config.example.json (which has idempotency semantics the existing code handles at line 350).
    - Do NOT copy `.git` or any other repo metadata — Pitfall 9 guard. The glob `*.md` already prevents this.
