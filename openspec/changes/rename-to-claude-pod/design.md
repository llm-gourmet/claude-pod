## Context

The project directory is already named `claude-pod` and the git remote reflects the new name. The source code, however, still uses `claude-secure` extensively: as the CLI binary name, config directory prefix, systemd unit prefix, container-internal path prefix, log path prefix, Python class attributes, Bash variable names, and in all documentation. This change brings the code into alignment with the repository name.

Affected surface area (66 non-openspec source files, ~907 unique lines):

| Area | Old | New |
|------|-----|-----|
| CLI binary | `bin/claude-secure`, `/usr/local/bin/claude-secure` | `bin/claude-pod`, `/usr/local/bin/claude-pod` |
| User config dir | `~/.claude-secure/` | `~/.claude-pod/` |
| Container config | `/etc/claude-secure/` | `/etc/claude-pod/` |
| Log path | `/var/log/claude-secure/` | `/var/log/claude-pod/` |
| Systemd units | `claude-secure-webhook`, `claude-secure-reaper` | `claude-pod-webhook`, `claude-pod-reaper` |
| Python attrs | `claude_secure_bin`, `claude_secure_bootstrap_path` | `claude_pod_bin`, `claude_pod_bootstrap_path` |
| Bash vars | `claude_secure_brew_prefix`, etc. | `claude_pod_brew_prefix`, etc. |

## Goals / Non-Goals

**Goals:**
- Every user-visible and code-visible reference to `claude-secure` / `claude_secure` becomes `claude-pod` / `claude_pod`
- All installed artifacts (binary, systemd units, config paths) use the new name after a fresh install
- Tests and fixtures pass with the new names

**Non-Goals:**
- Migrating existing user installs (covered by the uninstall command, not this change)
- Changing any functional behavior, architecture, or configuration format
- Updating openspec change artifacts from prior changes (those are historical)

## Decisions

**Mechanical sed rename, not manual edits**
There are ~66 files and ~900+ occurrences. Doing this by hand risks missing instances. The implementation task uses `sed -i` with both `claude-secure` (hyphen) and `claude_secure` (underscore) patterns applied across the file tree, excluding `.git/` and `openspec/changes/` (historical artifacts). After the sed pass, the binary is physically renamed.

**Exclude `openspec/changes/` from rename**
Prior change proposals and tasks reference `claude-secure` as the project name at the time they were written. Rewriting history in those files adds noise with no benefit. The `openspec/specs/` directory does get updated since specs are living reference documents.

**Rename `bin/claude-secure` → `bin/claude-pod` as a git mv**
Preserves history. The install script installs the binary to `/usr/local/bin/claude-pod`.

**Systemd unit file names**
Unit files on disk in `config/` or generated by the installer must change. The installer and uninstaller reference unit names as strings — these get updated by the sed pass.

**CLAUDE.md project name**
CLAUDE.md contains `claude-secure` as the project name in the header. It gets updated to `claude-pod`.

## Risks / Trade-offs

- **Existing installs break**: Users with `~/.claude-secure/` and `claude-secure-webhook` systemd units must uninstall and reinstall. The uninstall command handles removal of the old artifacts; the new install creates new ones.
- **Test fixture JSON files**: Files like `tests/fixtures/*/profile.json` may contain `claude_secure_bin` as a key name or value. These must be updated consistently or fixture-based tests will fail.
- **Missed occurrences**: Any file not covered by the `sed` glob patterns (e.g., `.txt`, `.conf`, `.service` template files) could be missed. A post-rename grep verifies zero remaining `claude-secure` / `claude_secure` occurrences outside `openspec/changes/`.
